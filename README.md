# Week 8 — 영화 애플리케이션

7주차에서 구현한 Props & Emit 기반 코드를 이어받아 옵션 A, B, C를 모두 추가로 구현했습니다.

## 구현 방식

**computed 활용**: 정렬 결과를 `sortedMovies`로 분리해 원본 배열을 직접 수정하지 않도록 했습니다. 같은 버튼을 다시 클릭하면 방향이 토글되고, 다른 기준으로 바꿀 때는 내림차순으로 초기화됩니다.

**컴포넌트 분리 및 단방향 데이터 흐름**: 영화 추가는 `AddMovieForm.vue`로 분리하고, 자식 컴포넌트는 `emit`으로만 부모와 통신합니다. 부모(App.vue)가 영화 데이터의 원본을 단독으로 관리합니다.

**인라인 유효성 처리**: 에러/성공 메시지를 브라우저 alert 없이 컴포넌트 내부에서 인라인으로 처리하고, 일정 시간 후 자동 소멸하도록 했습니다.

## 화면별 기능

### 1. 영화 목록
- 전체 영화를 카드 형태로 표시하며 좋아요 / 삭제가 가능합니다.
- 각 카드의 [수정] 버튼을 누르면 제목·평점을 바로 수정할 수 있는 인라인 편집 모드로 전환됩니다.
- 수정 모드인 카드는 테두리로 구분되며, [저장]으로 부모 데이터를 갱신하거나 [취소]로 원래 상태로 돌아올 수 있습니다.

| 뷰 모드 (기본) | 수정 모드 활성화 | 수정 에러 상태 |
|---|---|---|
| ![카드 뷰](screenshots/card-view.png) | ![카드 수정](screenshots/card-edit.png) | ![수정 에러](screenshots/card-edit-error.png) |

### 2. 정렬 바
- 최신순 / 평점 순 / 좋아요 순 세 가지 기준으로 실시간 정렬됩니다.
- 현재 활성화된 버튼은 파란색으로 구분되고, 같은 버튼을 다시 누르면 오름차순 ↔ 내림차순이 전환됩니다.

| 기본 상태 (최신순 ▼ 활성) | 평점 순 ▲ 오름차순 전환 |
|---|---|
| ![정렬 기본](screenshots/sort-default.png) | ![정렬 전환](screenshots/sort-asc.png) |

### 3. 영화 추가 폼
- 제목과 평점을 입력해 새 영화를 등록하면 목록에 즉시 반영됩니다.
- 빈값 제출이나 범위 초과 평점 입력 시 해당 필드에 에러 표시가 나타나고 3초 후 자동으로 사라집니다.
